## 1. 파이프라인별 LLM 호출 구조

| 단계 | 모델 | 역할 | Input tokens | Output tokens | 호출 수 |
|------|------|------|-------------|---------------|---------|
| Skeleton 생성 | Sonnet 4.6 | 도시배분/항공/호텔/가격 | ~9,000 | ~3,000 | 1회 |
| Skeleton 검증 | 코드 로직 | 구조 검증 | - | - | LLM 아님 |
| Day Detail 생성 | Opus 4.6 | 일자별 관광지/식사/트렌드 | ~10,000 | ~2,000 | **4회** (일수) |
| Day Detail 검증 | 코드 로직 | 관광지 수/도시 연결 검증 | - | - | LLM 아님 |

### Input tokens 구성

| 구성요소 | 토큰 수 | 비고 |
|----------|---------|------|
| System prompt (Skeleton) | ~2,100 | 고정 |
| System prompt (Day Detail) | ~2,700 | 고정 |
| Graph Context | ~7,000 | 참고상품 + 검색결과 + 노선 + 관광지 + 호텔 |
| 유사도 규칙 | ~300 | 5-Layer 규칙 텍스트 |
| 사용자 입력 | ~200 | 기획 요청 조건 |

|------|-----------|------------|------|

## 5. 변동 요인

| 요인 | 영향 | 비용 변화 |
|------|------|----------|
| **일수 증가** (5박6일) | Day Detail 6회 | ~$1.80 → +$0.50 |
| **일수 증가** (7박8일) | Day Detail 8회 | ~$2.40 → +$1.10 |
| **검증 실패 재시도** | Skeleton 또는 Day Detail 재생성 | 최대 2배 (~$2.60) |
| **Extended Thinking** | Opus 심층 사고 활성화 시 | +30~50% 추정 |
| **Prompt Caching (현행)** | System prompt + 도구 정의 캐시 재사용 | Opus 입력 -90%, Sonnet 입력 -90% |
| **캐시 히트** | 동일 지역 반복 기획 시 context 수집 절감 | -5% |
| **Sonnet으로 전환** | Day Detail을 Sonnet으로 대체 | ~$0.15 (90% 절감, 품질 저하) |
